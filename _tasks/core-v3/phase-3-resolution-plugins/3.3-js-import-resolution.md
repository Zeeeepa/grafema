# Task 3.3: JS Import Resolution Plugin

## Goal

Match IMPORT_BINDING nodes to export definitions across files. Handle ESM named/default imports, CJS require, re-exports. Emit IMPORTS_FROM edges.

## Files to Modify

- `packages/grafema-resolve/src/ImportResolution.hs`

## Reference

- Core-v3 plan constraint O2: cycle detection via visited set for re-export chains

## Subtasks

### 3.3a: Core algorithm
For each IMPORT_BINDING node:
1. Extract source module path and imported name
2. Find target module's exports (from ExportInfo in graph)
3. Match by name → emit IMPORTS_FROM edge
4. If not found → leave IMPORT_BINDING as unresolved

### 3.3b: ESM named imports
`import { foo } from './bar'`
- IMPORT_BINDING node has `importedName: "foo"`, `source: "./bar"`
- Find bar's ExportInfo where `name == "foo"` and `kind == "named"`
- Emit edge: IMPORT_BINDING → export's nodeId, type: IMPORTS_FROM

### 3.3c: ESM default imports
`import bar from './bar'`
- IMPORT_BINDING node has `importedName: "default"`, `localName: "bar"`
- Find bar's ExportInfo where `kind == "default"`
- Emit IMPORTS_FROM edge

### 3.3d: ESM namespace imports
`import * as bar from './bar'`
- IMPORT_BINDING node has `importedName: "*"`, `localName: "bar"`
- Create namespace binding node that references all of bar's exports
- Emit IMPORTS_FROM edge to module node

### 3.3e: CJS require
`const bar = require('./bar')`
- IMPORT_BINDING with `source: "./bar"`, `kind: "cjs"`
- Match to `module.exports` assignment in target file
- Emit IMPORTS_FROM edge

### 3.3f: Re-export chains
`export { foo } from './bar'` where bar re-exports from baz:
- Follow chain: current file → bar → baz → ... → find actual definition
- Cycle detection: maintain visited set of (module, name) pairs
- Max depth limit (e.g., 10) as safety net
- On cycle: emit ISSUE node, leave import unresolved

### 3.3g: Emit IMPORTS_FROM edges
For each resolved import:
```haskell
EmitEdge GraphEdge
  { geSource = importBindingNodeId
  , geTarget = exportNodeId
  , geType = "IMPORTS_FROM"
  , geMetadata = Map.fromList [("resolvedVia", "import-resolution")]
  }
```

### 3.3h: Module path resolution
Resolve module specifiers to file paths:
- Relative: `./foo` → try `foo.js`, `foo.ts`, `foo/index.js`, `foo/index.ts`
- Bare specifiers: `lodash` → `node_modules/lodash/` (follow package.json main/exports)
- Index files: `./utils` → `utils/index.js`
- TypeScript: `.ts`, `.tsx` extensions, path aliases from tsconfig

## Acceptance Criteria

- Given files A importing from B, correct IMPORTS_FROM edges appear in graph
- Named, default, namespace, and CJS imports all resolve
- Re-export chains followed correctly (with cycle detection)
- Module path resolution handles relative, bare, and index files
- Unresolvable imports produce clear diagnostic (not silent failure)
