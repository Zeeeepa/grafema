# Task 3.1: grafema-common Haskell Library

## Goal

Shared Haskell library with types used by both `grafema-analyzer` and `grafema-resolve`. Prevents type duplication and ensures protocol compatibility.

## Files to Create

- `packages/grafema-common/grafema-common.cabal`
- `packages/grafema-common/src/Grafema/Types.hs`
- `packages/grafema-common/src/Grafema/Protocol.hs`
- `packages/grafema-common/src/Grafema/Json.hs`

## Files to Modify

- `packages/grafema-analyzer/grafema-analyzer.cabal` (add dependency on grafema-common)

## Subtasks

### 3.1a: Cabal library package
```cabal
cabal-version: 3.0
name: grafema-common
version: 0.1.0
library
  exposed-modules:
    Grafema.Types
    Grafema.Protocol
    Grafema.Json
  build-depends: base, aeson, text, bytestring, containers
```

### 3.1b: Extract shared types
Move from grafema-analyzer to grafema-common:
- `GraphNode` — node with id, type, file, metadata
- `GraphEdge` — edge with source, target, type, metadata
- `MetaValue` — metadata value type (string, number, bool, null)
- `SemanticId` — newtype over Text for semantic IDs
- `ExportInfo`, `ExportKind` — from task 1.5

### 3.1c: Plugin protocol types
```haskell
-- Input: what the orchestrator sends to a streaming plugin
data PluginInput = PluginInput
  { piNodes :: [GraphNode]
  }

-- Output: what a plugin emits
data PluginCommand
  = EmitNode GraphNode
  | EmitEdge GraphEdge
```
NDJSON format: one JSON object per line on stdin/stdout.

### 3.1d: JSON encoding/decoding
Shared `ToJSON`/`FromJSON` instances for all protocol types. Single source of truth for JSON format — both analyzer and plugins use the same instances.

### 3.1e: Update grafema-analyzer dependency
- Add `grafema-common` as dependency
- Replace local type definitions with imports from `Grafema.Types`
- Verify `cabal build` still works

## Acceptance Criteria

- `grafema-common` builds as a library
- `grafema-analyzer` imports types from `grafema-common` (no local duplicates)
- `grafema-resolve` (task 3.2) can import the same types
- JSON encoding is consistent between all consumers

## Dependencies

None for creation. Updating grafema-analyzer depends on types being stable (after Phase 1 tasks).

## Estimated Scope

~0.5 day. Extract and reorganize existing types.
