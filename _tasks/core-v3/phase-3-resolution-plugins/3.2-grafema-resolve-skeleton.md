# Task 3.2: grafema-resolve Skeleton

## Goal

New Haskell binary that implements the plugin protocol. Subcommands for each resolution plugin. Start with streaming mode (simpler), migrate to batch if performance requires.

## Files to Create

- `packages/grafema-resolve/grafema-resolve.cabal`
- `packages/grafema-resolve/src/Main.hs`
- `packages/grafema-resolve/src/ImportResolution.hs` (stub)
- `packages/grafema-resolve/src/RuntimeGlobals.hs` (stub)

## Subtasks

### 3.2b: Main.hs with subcommand dispatch
```haskell
data Command
  = Imports   -- JS import resolution
  | RuntimeGlobals  -- Runtime global matching

main :: IO ()
main = do
  cmd <- execParser opts
  case cmd of
    Imports -> ImportResolution.run
    RuntimeGlobals -> RuntimeGlobals.run
```

CLI: `grafema-resolve imports`, `grafema-resolve runtime-globals`

### 3.2c: Streaming mode protocol
Read NDJSON from stdin (nodes from orchestrator query), process, write NDJSON to stdout:
```haskell
streamingMain :: (GraphNode -> [PluginCommand]) -> IO ()
streamingMain process = do
  input <- BL.getContents
  let nodes = map decodeNode (BL.lines input)
  let commands = concatMap process nodes
  mapM_ (BL.putStrLn . encode) commands
```

### 3.2d: Batch mode protocol (future)
Placeholder for connecting to RFDB directly:
- Read `RFDB_SOCKET` env var
- MessagePack client for Haskell (use `msgpack` package)
- Query RFDB, process, write results back

Decision: **start with streaming mode**. Batch mode adds complexity (Haskell RFDB client) and is only needed if streaming becomes a bottleneck.
