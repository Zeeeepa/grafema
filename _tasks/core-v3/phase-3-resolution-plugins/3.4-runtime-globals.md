# Task 3.4: Runtime Globals Plugin

## Goal

Match unresolved REFERENCE nodes (`resolved: false`) to runtime/global definitions (console, process, window, etc.). Emit RESOLVES_TO edges for matched references.

## Files to Modify

- `packages/grafema-resolve/src/RuntimeGlobals.hs`

## Subtasks

### 3.4a: Built-in globals database
Define known globals as a static database:

**Node.js globals**: `console`, `process`, `Buffer`, `__dirname`, `__filename`, `module`, `exports`, `require`, `global`, `setTimeout`, `setInterval`, `setImmediate`, `clearTimeout`, `clearInterval`, `clearImmediate`, `queueMicrotask`

**Browser globals**: `window`, `document`, `navigator`, `location`, `history`, `localStorage`, `sessionStorage`, `fetch`, `XMLHttpRequest`, `WebSocket`, `requestAnimationFrame`, `cancelAnimationFrame`, `alert`, `confirm`, `prompt`

**Shared globals**: `JSON`, `Math`, `Date`, `Promise`, `Array`, `Object`, `String`, `Number`, `Boolean`, `Symbol`, `Map`, `Set`, `WeakMap`, `WeakSet`, `Proxy`, `Reflect`, `Error`, `TypeError`, `RangeError`, `RegExp`, `parseInt`, `parseFloat`, `isNaN`, `isFinite`, `encodeURIComponent`, `decodeURIComponent`, `undefined`, `NaN`, `Infinity`

Structure:
```haskell
data GlobalDef = GlobalDef
  { gdName :: Text
  , gdCategory :: GlobalCategory  -- NodeJs | Browser | EcmaScript
  , gdType :: Text                 -- "function" | "object" | "class" | "constant"
  }
```

### 3.4b: Query REFERENCE nodes
Input: stream of REFERENCE nodes where `resolved == false`.
Filter from orchestrator query: `find_nodes(type: "REFERENCE", metadata: {resolved: false})`

### 3.4c: Match against globals database
For each unresolved REFERENCE:
- Extract reference name
- Look up in globals database (O(1) HashMap lookup)
- If found → mark for resolution

### 3.4d: Emit RESOLVES_TO edges
For matched references:
```haskell
EmitEdge GraphEdge
  { geSource = referenceNodeId
  , geTarget = globalDefNodeId  -- virtual node for the global
  , geType = "RESOLVES_TO"
  , geMetadata = Map.fromList
      [ ("resolvedVia", "runtime-globals")
      , ("globalCategory", category)
      ]
  }
```

Also emit virtual GLOBAL_DEFINITION nodes for matched globals (if not already in graph).

### 3.4e: Unresolved remainder
References not matching any global remain as REFERENCE(`resolved: false`):
- These represent genuinely unknown identifiers
- Could be: framework-injected, dynamic, or actual bugs in user code
- Visible in graph for user inspection / future plugins

## Acceptance Criteria

- `console.log()` reference gets RESOLVES_TO edge to console global
- `process.env` reference gets RESOLVES_TO edge to process global
- `Math.floor()` reference resolves
- Unknown `myGlobal` stays as REFERENCE(resolved:false)
- No false positives (local variables named `console` shouldn't match if resolved by scope)

## Dependencies

- **3.2** (grafema-resolve skeleton with plugin protocol)

## Estimated Scope

~0.5 day. Globals database is a static list. Matching is simple HashMap lookup.
