# Task 2.2: RFDB Client

## Goal

MessagePack client in Rust that connects to RFDB unix socket and implements protocol v3 commands needed by the orchestrator.

## Files to Modify

- `packages/grafema-orchestrator/src/rfdb.rs`

## Key References

- `packages/rfdb-server/src/session.rs` — protocol implementation (Rust)
- `packages/rfdb/src/index.ts` — existing JS client
- RFDB protocol v3: length-prefixed MessagePack frames over unix socket

## Subtasks

### 2.2a: Unix socket connection + MessagePack framing
- Connect to unix socket path
- Length-prefixed framing: 4-byte big-endian length prefix, then MessagePack payload
- Async I/O via tokio

### 2.2b: Protocol negotiation
- Send handshake with protocol version 3
- Validate server response

### 2.2c: OpenDatabase / CreateDatabase
- `OpenDatabase { name }` — open existing database
- `CreateDatabase { name }` — create new database
- Handle "already exists" gracefully

### 2.2d: BeginBatch / AddNode / AddEdge / CommitBatch
- `BeginBatch` — start a write batch
- `AddNode { id, type, file, metadata }` — add node to batch
- `AddEdge { source, target, type, metadata }` — add edge to batch
- `CommitBatch { changedFiles }` — commit batch, trigger GC for listed files

### 2.2e: Query command
- `Query { query, params }` — execute Datalog query
- Return results as `Vec<HashMap<String, Value>>`
- Needed by plugins that read graph state

### 2.2f: SemanticId → u128 conversion
- Haskell emits semantic IDs as strings (e.g., `"file.js::function::foo"`)
- Orchestrator converts to u128 via BLAKE3 hash before storing in RFDB
- Implement: `fn semantic_id_to_u128(id: &str) -> u128 { blake3::hash(id.as_bytes()).as_bytes()[..16] }`

### 2.2g: Error handling and reconnection
- Typed error enum for RFDB errors
- Automatic reconnection on socket disconnect
- Timeout handling for unresponsive server

## Acceptance Criteria

- Can connect to running `rfdb-server`
- Can create database, add nodes/edges via batch, query them back
- SemanticId string → u128 conversion matches expected values
- Handles server disconnection gracefully

## Dependencies

- **2.1** (crate must exist)

## Estimated Scope

~1 day. Protocol implementation requires careful alignment with rfdb-server.
