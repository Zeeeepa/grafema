# Task 2.5: Analysis Spawning

## Goal

For each discovered file: parse with OXC → pipe JSON to `grafema-analyzer` → read FileAnalysis JSON → ingest into RFDB.

## Files to Modify

- `packages/grafema-orchestrator/src/analyzer.rs`

## Subtasks

### 2.5a: Spawn grafema-analyzer
```rust
pub async fn analyze_file(file: &Path, ast_json: &str) -> Result<FileAnalysis> {
    let mut child = Command::new("grafema-analyzer")
        .arg(file.to_str().unwrap())
        .stdin(Stdio::piped())
        .stdout(Stdio::piped())
        .stderr(Stdio::piped())
        .spawn()?;

    child.stdin.take().unwrap().write_all(ast_json.as_bytes()).await?;
    let output = child.wait_with_output().await?;
    serde_json::from_slice(&output.stdout)?
}
```

### 2.5b: Parse FileAnalysis response
Deserialize `{ nodes, edges, exports }` JSON from analyzer stdout:
```rust
#[derive(Deserialize)]
pub struct FileAnalysis {
    pub nodes: Vec<GraphNode>,
    pub edges: Vec<GraphEdge>,
    pub exports: Vec<ExportInfo>,
}
```

### 2.5c: Convert and ingest into RFDB
- Convert semantic ID strings to u128 via BLAKE3 (using rfdb client from 2.2f)
- Begin batch for the file
- AddNode for each node, AddEdge for each edge
- CommitBatch with `changedFiles: [file_path]`

### 2.5d: Parallel execution
Bounded worker pool using tokio semaphore or rayon:
```rust
let semaphore = Arc::new(Semaphore::new(num_cpus));
let handles: Vec<_> = files.iter().map(|f| {
    let sem = semaphore.clone();
    tokio::spawn(async move {
        let _permit = sem.acquire().await;
        analyze_and_ingest(f).await
    })
}).collect();
```

### 2.5e: Progress reporting
- Print `[42/100] Analyzing src/foo.js...` style progress
- Summary at end: `Analyzed 100 files: 4523 nodes, 8901 edges, 3 errors`

### 2.5f: Error handling
- If analyzer process exits non-zero: emit ISSUE node for the file, continue
- If analyzer produces invalid JSON: emit ISSUE node, continue
- Collect all errors, report summary at end
- Never fail the entire run because of one file

## Acceptance Criteria

- Can analyze a directory of JS files, all nodes/edges appear in RFDB
- Parallel execution with configurable concurrency
- Progress reporting shows current file and overall progress
- Error in one file doesn't stop analysis of other files
