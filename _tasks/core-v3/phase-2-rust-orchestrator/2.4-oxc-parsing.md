# Task 2.4: OXC Parsing

## Goal

Use OXC Rust crate to parse JS/TS files into ESTree JSON that `grafema-analyzer` expects on stdin.

## Files to Modify

- `packages/grafema-orchestrator/Cargo.toml` (add OXC dependencies)
- `packages/grafema-orchestrator/src/parser.rs`

## Subtasks

### 2.4a: Add OXC dependencies
```toml
oxc_parser = "0.x"
oxc_ast = "0.x"
oxc_allocator = "0.x"
oxc_span = "0.x"
```
Pin to a specific version that produces ESTree-compatible JSON.

### 2.4b: Parse → ESTree JSON
```rust
pub fn parse_to_estree(source: &str, filename: &str) -> Result<String> {
    let allocator = Allocator::default();
    let source_type = SourceType::from_path(filename)?;
    let ret = Parser::new(&allocator, source, source_type).parse();
    // Serialize to ESTree JSON format
    serde_json::to_string(&ret.program)?
}
```

Verify output matches what `grafema-analyzer` currently expects. Current core-v2 uses Node.js OXC wrapper — diff the output formats.

### 2.4c: Handle parse errors
- If OXC returns parse errors: emit ISSUE node for the file (type=PARSE_ERROR), continue
- Partial ASTs: if OXC produces a partial AST with errors, still process it
- Log parse errors with file path and error details

### 2.4d: Parallel parsing
Use rayon or `tokio::spawn_blocking` for parallel file parsing:
```rust
pub fn parse_files(files: &[PathBuf], jobs: usize) -> Vec<ParseResult> {
    files.par_iter()
        .map(|f| parse_file(f))
        .collect()
}
```

## Acceptance Criteria

- Can parse a `.js` file and produce JSON that `grafema-analyzer` accepts on stdin
- TypeScript files (`.ts`, `.tsx`) parse correctly
- JSX/TSX syntax handled
- Parse errors don't crash — produce ISSUE node and continue
- Parallel parsing works with configurable worker count
