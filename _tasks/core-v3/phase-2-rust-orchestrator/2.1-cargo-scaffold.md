### 2.1b: CLI with clap


### 2.1c: Module stubs
Each module file gets a stub with the public interface documented:
- `config.rs` — config parsing (task 2.3)
- `discovery.rs` — file discovery (task 2.3)
- `parser.rs` — OXC parsing (task 2.4)
- `rfdb.rs` — RFDB client (task 2.2)
- `analyzer.rs` — analysis spawning (task 2.5)
- `plugin.rs` — plugin DAG runner (task 2.6)
- `gc.rs` — generation GC (task 2.7)

- `cargo build` succeeds

- Project structure is clean and ready for parallel development of modules
