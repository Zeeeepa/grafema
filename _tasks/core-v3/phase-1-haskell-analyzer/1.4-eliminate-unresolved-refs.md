# Task 1.4: Eliminate unresolvedRefs from External Output

## Goal

After `resolveFileRefs` is working, `DeferredRef` becomes purely internal. Remove from `FileAnalysis` external contract. Clean external output: `{ nodes, edges, exports }`.

## Files to Modify

- `Analysis/Types.hs`
- `Analysis/Context.hs`
- `Output/Encode.hs`

## Subtasks

### 1.4a: Split FileAnalysis into internal and external types
Internal type (used during analysis):
```haskell
data InternalFileAnalysis = InternalFileAnalysis
  { ifaNodes :: [GraphNode]
  , ifaEdges :: [GraphEdge]
  , ifaExports :: [ExportInfo]
  , ifaDeferredRefs :: [DeferredRef]  -- internal only
  }
```

External type (serialized to JSON):
```haskell
data FileAnalysis = FileAnalysis
  { faNodes :: [GraphNode]
  , faEdges :: [GraphEdge]
  , faExports :: [ExportInfo]
  }
```

### 1.4b: Internal type keeps DeferredRefs
`InternalFileAnalysis` keeps `ifaDeferredRefs` for `resolveFileRefs` input. The resolver consumes deferred refs and produces edges/updated nodes.

### 1.4c: External output structure
JSON output has exactly three fields: `nodes`, `edges`, `exports`. No `unresolvedRefs`.

### 1.4d: Update ToJSON instance
`ToJSON FileAnalysis` encodes only `nodes`, `edges`, `exports`. Remove `unresolvedRefs` from JSON encoding.

### 1.4e: Clean up serialization
`DeferredRef` and `DeferredKind` types stay in source (used internally by resolver), but are never serialized to JSON. Remove their `ToJSON` instances if they exist.

## Acceptance Criteria

- `grafema-analyzer` JSON output has **no** `unresolvedRefs` field
- Output contains only `nodes`, `edges`, `exports`
- Internal analysis pipeline still uses DeferredRef for scope resolution
- No breaking changes to DeferredRef/DeferredKind types themselves

## Dependencies

- **1.3** (resolveFileRefs must be working before removing unresolvedRefs from output)

## Estimated Scope

~0.5 day. Type refactoring and JSON encoding changes.
