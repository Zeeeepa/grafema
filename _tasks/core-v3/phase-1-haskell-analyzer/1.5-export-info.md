# Task 1.5: Add ExportInfo to FileAnalysis

## Goal

`FileAnalysis` needs `faExports :: [ExportInfo]` so the orchestrator and resolution plugins know what each file exports. Required for cross-file import resolution (Phase 3).

## Files to Modify

- `Analysis/Types.hs`
- `Rules/Declarations.hs`
- `Output/Encode.hs`

## Subtasks

### 1.5a: Define ExportInfo type
```haskell
data ExportKind = NamedExport | DefaultExport | ReExport
  deriving (Show, Eq, Generic)

data ExportInfo = ExportInfo
  { eiName    :: Text       -- exported name (or "default")
  , eiNodeId  :: Text       -- semantic ID of the exported node
  , eiKind    :: ExportKind
  , eiSource  :: Maybe Text -- for re-exports: source module specifier
  }
  deriving (Show, Eq, Generic)
```

### 1.5b: Collect exports from rules
- `ruleExportDeclaration` → NamedExport for each specifier, or for the declared entity
- `ruleExportDefaultDeclaration` → DefaultExport with name "default"
- `ruleExportAllDeclaration` (`export * from '...'`) → ReExport with source module

Collect into `faExports` field on FileAnalysis during analysis.

### 1.5c: Add ToJSON for ExportInfo
```json
{
  "name": "foo",
  "nodeId": "file.js::function::foo",
  "kind": "named",
  "source": null
}
```

`ExportKind` serializes as lowercase string: `"named"`, `"default"`, `"reexport"`.

### 1.5d: Include in external FileAnalysis output
Add `exports` field to FileAnalysis JSON output alongside `nodes` and `edges`.

## Acceptance Criteria

- `export function foo()` → ExportInfo `{name: "foo", nodeId: "...", kind: "named"}`
- `export default class Bar` → ExportInfo `{name: "default", nodeId: "...", kind: "default"}`
- `export { baz } from './qux'` → ExportInfo `{name: "baz", nodeId: "...", kind: "reexport", source: "./qux"}`
- `export * from './utils'` → ExportInfo `{name: "*", nodeId: null, kind: "reexport", source: "./utils"}`
- Exports appear in `grafema-analyzer` JSON output under `exports` key
