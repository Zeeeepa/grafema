# Task 1.2: Populate drScopeId on DeferredRef

## Goal

Every `emitDeferred` call passes `drScopeId = Just <current scope id>` instead of `Nothing`. This is needed for `resolveFileRefs` (task 1.3) to know where to start the scope chain walk when resolving references.

## Files to Modify

- `Rules/Expressions.hs`
- `Rules/Declarations.hs`
- `Rules/TypeScript.hs`

## Subtasks

### 1.2a: ruleCallExpression
**File**: `Rules/Expressions.hs` (around line 68)
Add `askScopeId` before `emitDeferred` to capture current scope. Pass as `drScopeId = Just scopeId`.

### 1.2b: ruleIdentifier
**File**: `Rules/Expressions.hs` (around line 229)
Add scope ID to deferred ref for standalone identifier references.

### 1.2c: ruleMemberExpression chain
**File**: `Rules/Expressions.hs` (around line 518)
Add scope ID to member expression deferred refs. The scope is where the root identifier of the chain appears.

### 1.2d: ruleImportDeclaration
**File**: `Rules/Declarations.hs` (around lines 298, 456)
Add scope ID for import-related deferred refs. These will be ImportResolve kind (cross-file), but still need scope context for potential shadowing analysis.

### 1.2e: ruleTypeReference
**File**: `Rules/TypeScript.hs` (around line 154)
Add scope ID for TypeScript type references.

## Acceptance Criteria

- **No** `DeferredRef` has `drScopeId = Nothing`
- Each deferred ref has the scope ID of the scope where the reference textually appears
- `askScopeId` is used consistently (not hardcoded scope IDs)
