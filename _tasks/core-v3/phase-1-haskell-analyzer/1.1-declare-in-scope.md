# Task 1.1: Wire up declareInScope Calls

## Goal

Every declaration rule calls `declareInScope` so the scope chain tracks what's declared where. Currently `declareInScope` exists in `Analysis/Scope.hs` but is never called — scopes are created but never populated.

## Files to Modify

- `Rules/Declarations.hs`
- `Rules/Statements.hs`
- `Rules/Patterns.hs`

## Subtasks

### 1.1a: ruleVariableDeclaration
**File**: `Rules/Declarations.hs`
Wrap continuation in `declareInScope` for each declarator. Each `VariableDeclarator` with an `id` (Identifier or pattern) must register in the current scope. `var` → function scope, `let`/`const` → block scope.

### 1.1b: ruleFunctionDeclaration
**File**: `Rules/Declarations.hs`
Declare function name in **parent** scope (function declarations are visible in enclosing scope). Parameters are declared in the **function** scope (handled by 1.1d).

### 1.1c: ruleClassDeclaration
**File**: `Rules/Declarations.hs`
Declare class name in **parent** scope. Class body creates its own scope for methods/properties, but the class name binding goes in the enclosing scope.

### 1.1d: ruleParam
**File**: `Rules/Patterns.hs`
Declare parameter name in current scope (which should be the function scope). Handle destructuring patterns — each binding identifier in the pattern gets declared.

### 1.1e: ruleCatchClause
**File**: `Rules/Statements.hs`
Declare catch binding (e.g., `catch (e) { ... }`) in the catch clause's block scope.

### 1.1f: ruleImportDeclaration
**File**: `Rules/Declarations.hs`
Declare imported names as `DeclImport` in module scope. Each ImportSpecifier/ImportDefaultSpecifier/ImportNamespaceSpecifier creates a declaration for the local binding name.

## Acceptance Criteria

- Every `GraphNode` of type FUNCTION/VARIABLE/CLASS/PARAMETER has a corresponding `Declaration` in the scope where it's visible
- `declareInScope` is called at every declaration site
- `var` declarations are registered in the nearest function scope
- `let`/`const` declarations are registered in the nearest block scope
- Function declarations are hoisted to parent scope

## Dependencies

None — can start immediately.

## Estimated Scope

~0.5 day. Mechanical wiring: each rule already has the scope context, just needs `declareInScope` calls.
